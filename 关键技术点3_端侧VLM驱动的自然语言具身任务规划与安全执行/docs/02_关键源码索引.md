# 关键源码索引

## 展示范围说明

本目录整理第三个关键技术点中能够体现通用算法链路的核心源码：自然语言解析、端侧 VLM 调用、结构化任务表示、技能映射、语义目标裁判、语音反馈和任务报告。

## 源码分组

| 模块 | 文件 | 作用 |
|---|---|---|
| 语音与文本入口 | `voice_input_node.py` | 将语音识别结果接入自然语言任务链路 |
| 自然语言解析入口 | `nl_parser_node.py` | 组织 VLM Planner、Validator 和 Executor，形成任务解析主链路 |
| VLM 调用封装 | `vlm_client.py`, `vlm_planner.py` | 调用端侧 VLM 服务，将自然语言输入转为结构化语义结果 |
| 提示词与解析 | `nl_prompt.py`, `vlm_intent_parser.py` | 约束 VLM 输出 JSON，并解析为本地中间表示 |
| 结构化任务表示 | `nlu_schema.py` | 定义 `ParsedIntent`、`SemanticStep`、`StageAction` 等中间对象 |
| 多步任务处理 | `task_segmenter.py` | 对复合自然语言任务进行保守切分，避免误拆语义描述 |
| 本地安全约束 | `intent_validator.py`, `skill_mapper.py` | 将 VLM steps 约束到本地可执行技能和合法参数范围 |
| 统一执行出口 | `task_executor.py` | 统一控制任务下发，保证回复、取消和执行顺序可控 |
| 开放语义裁判 | `semantic_match_prompt.py`, `person_semantic_matcher_node.py`, `semantic_target_matcher_node.py` | YOLO 候选编号、VLM 三阶段视觉事实观察/条件拆解/候选裁判 |
| 场景反馈 | `scene_narrator_node.py`, `tts_node.py` | 生成场景解说和语音反馈 |
| 任务记录 | `mission_logger_node.py` | 记录任务过程、关键帧和结构化报告 |

## 配置文件

| 文件 | 作用 |
|---|---|
| `semantic_target_matcher_params.yaml` | 语义目标匹配节点参数，包括 VLM 接口、候选阈值和结果话题 |
| `scene_narrator_params.yaml` | 场景解说节点参数 |
| `voice_input_params.yaml` | 语音输入节点参数 |
| `tts_params.yaml` | 语音反馈节点参数 |
| `mission_logger_params.yaml` | 任务日志与报告生成参数 |
